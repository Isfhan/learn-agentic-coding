Every era of software rewrote the job.

Each new layer automated what the last generation did by hand. The developers who figured out the layer first became the ones everyone else relied on. Agents are the current layer. Your job is to learn to direct them.

The big picture, in 6 ideas

Internalize these. Everything else is commentary.

Strip the mystique. Most of the "AI engineering" field collapses into these six ideas. If you actually hold them in your head, you'll save months of confusion.

01

Agents are just LLMs in a loop

An agent = while not done: pick_tool(); run(); feed_back(). That's it. The rest is plumbing.

02

Context is the product

What you put into the window matters more than the model or the prompt. Context engineering is the skill.

03

MCP is the USB-C of AI

One standard. Any client (Cursor, Claude, Copilot) plugs into any server (GitHub, Postgres, your API).

04

Skills > Prompts > Rules

Prompts are per-task. Rules per-repo. Skills are per-capability and travel with you. Scale accordingly.

05

Evals beat vibes

"It felt better" is not a shipping criterion. Write evals early. Your future self will thank you.

06

Spec is the source of truth

You write the spec. The agent generates code. Code is the build artifact — the spec is what you maintain.
